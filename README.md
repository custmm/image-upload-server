## 프로젝트 소개

개인 제작물 아카이브를 기반으로, 콘텐츠 분류 구조와 UI 인터랙션, 반응형 설계를 실험하기 위해 제작한 웹사이트입니다.
단순 결과물을 보여주는 것이 아니라, 구조 설계와 사용자 경험까지 함께 기록하는 것을 목표로 합니다.

## 프로젝트 특징

- 아카이브형 웹사이트 구조 설계
- 인터랙션 중심 UI/UX 구현
- LLM을 활용한 개발 프로세스 개선

이 프로젝트는 단순 코드 생성이 아닌,  문제 정의 → 구조 설계 → 디버깅 → 개선 과정에서 LLM을 보조 도구로 활용하여 완성되었습니다.

## LLM 활용 사례

### 1. 프론트엔드 상태 관리 개선
- 문제: preview.js의 역할이 과도하게 집중됨
- 해결: 기능 단위로 로직 분리 (sidebar / pagination / indicator)
- 결과: 가독성 및 유지보수성 향상

### 2. 사이드바 구조 설계
- 문제: 계층형 메뉴 구성 필요
- 해결: 토글 기반 UI 및 상태 관리 구조 설계
- 결과: 확장 가능한 메뉴 구조 구현

### 3. Git 충돌 해결
- 문제: non-fast-forward 충돌 발생
- 해결: rebase 흐름 정리 및 복구 과정 분석
- 결과: Git 작업 흐름 이해도 향상

### 4. UI/UX 개선
- 문제: 사용자 흐름이 직관적이지 않음
- 해결: UX 피드백 기반 개선
- 결과: 인터랙션 자연스러움 향상

## 기술 스택

### Frontend
- HTML5
- CSS3 (Flexbox, Dark/Light Mode)
- Vanilla JavaScript

### Backend
- Node.js
- Express
- Multer (이미지 업로드 처리)


  ## 주요 기능

### 이미지 관리
- 이미지 업로드 및 서버 저장
- 카테고리 기반 자동 분류

### 갤러리 시스템
- 카테고리 / 서브카테고리 구조
- 페이지네이션
- 미리보기 기능

### UI / UX
- 다크 모드
- 사이드바 메뉴
- 로딩 인디케이터
- 반응형 레이아웃

### 인터랙션 요소
- 숨겨진 기능 페이지
- 웹 미니 게임
