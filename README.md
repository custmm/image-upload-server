
개인 제작물 기록 과정에서 콘텐츠 분류, 시각적 식별, 반응형 구조를 검증하기 위해 제작한 웹사이트의 소스들을 저장하는 장소입니다.

이 사이트의 차별점은 LLM언어모델을 통해 아카이브형 웹사이트를 제작할 수 있다는걸 보여주기 위함입니다.

## LLM 활용 방식

이 프로젝트는 단순 코드 자동 생성이 아니라,
문제 정의 → 구조 설계 → 디버깅 → 개선 과정에서
LLM을 보조 도구로 활용했다.

[1. 복잡한 프론트엔드 상태 관리]
- 문제: preview.js가 과도하게 비대해지고 역할이 섞임
- LLM 활용:
  - 기능 분리 기준 제안
  - sidebar / pagination / indicator 로직 분리 방향 논의
- 결과:
  - UI 로직 가독성 개선
  - 수정 범위 예측 가능해짐

[2. 사이드바 메뉴 구조 설계]
- 문제: 게시판 / 부가기능 메뉴를 계층적으로 구성해야 했음
- LLM 활용:
  - 토글 기반 메뉴 구조 설계
  - DOM 상태 관리 방식 정리
- 결과:
  - 확장 가능한 사이드바 구조 완성

[3. Git 충돌 및 rebase 문제 해결]
- 문제: 원격 저장소와 로컬 브랜치 간 non-fast-forward 충돌
- LLM 활용:
  - 현재 Git 상태 해석
  - rebase 진행 상황 정리 및 복구 절차 정리
- 결과:
  - 충돌 원인 이해
  - Git 작업 흐름에 대한 이해도 향상

[4. UI/UX 디테일 개선]
- 문제: 사용자 인터랙션이 직관적이지 않음
- LLM 활용:
  - 시니어 FE 관점의 UX 피드백 요청
  - 개선 우선순위 정리
- 결과:
  - 사용자 흐름이 자연스러워짐

Ⅲ.모바일 앱 연동 (웹 게임 기반)

	-웹과 모바일 앱은 동일한 게임 로직을 공유
	-모바일 환경에 맞게 UI 및 조작 방식 최적화
	-앱 전용 기능 및 게임 모드 추가
